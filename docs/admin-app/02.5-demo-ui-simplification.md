# Phase 2.5 — Demo UI Simplification Plan

## Objective

Сохранить ощущение зрелого B2B POS, но убрать перегрузку и риски overengineering перед Фазой 3.

---

## 1) Complexity audit

## Is top bar overloaded?

### Current risk
Да, потенциально перегружен (shift + network + NFC + queue + operator + guest context + title).

### Simplification decision
- Оставить всегда видимыми только 4 вещи:
  1. Операция/экран
  2. Guest context (или “не выбран”)
  3. Shift status
  4. NFC/Network aggregated status
- `Unsynced queue` скрыть в popover/details, показывать только если `> 0`.

## Are we introducing too many states?

### Current risk
Да, если показывать full matrix online/offline/degraded + device + queue + incident flags одновременно.

### Simplification decision
- Свести к 3 публичным global states:
  - `Ready`
  - `Attention`
  - `Blocked`
- Детали состояния раскрывать по клику “Статус системы”.

## Are advanced features too early?

### Current risk
Да: undo таймер, расширенный incident центр, сложный reissue workflow могут съесть график.

### Simplification decision
- Для demo-core оставить:
  - top-up success + fallback action,
  - basic incident shortcut,
  - block card (simple path).
- Полный undo/reissue automation — только Tier B при наличии времени.

---

## 2) What to hide/collapse for demo

## Collapse in UI

1. Investor-specific analytics (secondary tab or hidden by role).
2. Deep technical status fields (raw IDs, diagnostics) — в debug/details only.
3. Secondary navigation options, не влияющие на demo narrative.

## Show only conditionally

1. Unsynced queue badge — only if `count > 0`.
2. Incident warning chips — only when active incident exists.
3. Extended card metadata — only in detail drawer.

---

## 3) Flow simplification decisions

## Guest onboarding
- Keep required fields minimal (name + phone).
- Avoid long forms/documents for demo.

## Top-up
- Use fixed preset list + manual keypad; no complex payment split.
- Single payment method visible in MVP (`cash`) with room for extension.

## Balance/history
- Show last 5–10 events max.
- No complex filtering in demo core.

## Card controls
- Phase 3 core: block card + clear status change.
- Reissue with transfer is optional enhancer.

## Shift
- Keep simple open/close flow + quick summary.
- Advanced reconciliation rules defer.

---

## 4) Advanced recovery scenarios to defer

Defer to backlog unless trivially safe:
- Multi-step rollback orchestration.
- Automatic conflict reconciliation for offline queue.
- SLA-driven incident workflows.
- Multi-operator approval chains.

---

## 5) Demo narrative protection

To prevent “too much happening at once”:
- One main CTA per screen.
- One highlighted metric per panel.
- One fallback action in errors (plus optional “more”).
- Keep success/error language operational, not technical.

---

## 6) Final simplification snapshot (frozen)

### Keep
- Clean shell
- Guest context
- Top-up core flow
- Balance + recent history
- Shift open/close basic
- Basic error recovery

### Defer
- Complex undo
- Full reissue automation
- Full queue management UI
- Advanced incident center


- Backlog: **Issue Card during Open Visit flow** (separate step after M3.5).
